# Mall Lite

## 一、项目简介

`mall-lite` 是一个用于学习和实践的轻量级电商系统，目标是从 **单体分层架构** 起步，逐步沉淀用户、商品、库存、订单、支付等核心业务能力，并为后续平滑拆分为微服务预留边界。

这个项目不追求一开始就堆满复杂组件，而是优先把电商核心链路做清楚：

1. 用户登录与地址管理。
2. 商品分类、SPU、SKU 管理。
3. 库存查询、锁定、扣减、释放。
4. 订单创建、取消、支付状态流转。
5. 模拟支付与回调处理。

## 二、项目定位

### 1. 学习目标

1. 掌握 Spring Boot 后端项目的标准分层设计。
2. 理解电商系统中订单、库存、支付之间的业务协作。
3. 练习 MySQL 表设计、事务控制、并发处理和缓存使用。
4. 为后续学习 Spring Cloud、消息队列、分布式事务打基础。

### 2. 设计原则

1. **先单体，后微服务**：前期用 Maven 多模块组织代码，降低启动成本。
2. **业务边界清晰**：按用户、商品、库存、订单、支付拆分模块。
3. **核心链路优先**：先完成下单主流程，再补后台管理、营销等扩展能力。
4. **文档同步沉淀**：数据库设计、架构设计和学习笔记都放在 `docs` 目录。

## 四、模块说明

| 模块 | 说明 |
| --- | --- |
| `mall-common` | 通用返回对象、异常处理、枚举、常量、工具类 |
| `mall-user` | 用户、登录、收货地址 |
| `mall-product` | 商品分类、SPU、SKU、商品上下架 |

| `mall-app` | Spring Boot 启动入口，聚合各业务模块 |

## 五、核心业务流程

### 1. 下单流程

1. 用户选择商品 SKU 和购买数量。
2. 系统校验用户状态、商品状态和库存数量。
3. 库存模块锁定库存。
4. 订单模块创建待支付订单。
5. 用户完成模拟支付。
6. 支付模块更新支付流水。
7. 订单模块更新订单状态。
8. 库存模块扣减真实库存。

### 2. 取消订单流程

1. 用户主动取消订单，或订单超时未支付。
2. 订单模块校验订单是否允许取消。
3. 订单状态变更为已取消。
4. 库存模块释放已锁定库存。
