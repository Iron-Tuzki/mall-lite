# 前台首页与商品浏览设计

## 一、目标

前台首页面向普通客户，优先承载商品浏览、商品详情、登录入口、购物车入口和后续秒杀活动入口。当前阶段先完成首页静态布局和商品详情页基础结构，再逐步接入后端商品接口、Redis 商品详情缓存和 OpenAPI 前端类型生成。

## 二、技术选型

1. Vue 3：前台页面框架。
2. Vite：前端构建工具。
3. TypeScript：约束接口返回结构，减少字段拼写错误。
4. Element Plus：提供按钮、输入框、标签、骨架屏、消息提示等基础组件。
5. Vue Router：管理首页、商品详情页、购物车页等路由。
6. Pinia：后续承载登录用户、购物车、全局状态。
7. Axios：封装 HTTP 请求。
8. OpenAPI：后续根据后端 `/v3/api-docs` 生成接口类型，减少前后端字段不一致。

## 三、页面布局

首页采用类似淘宝首页的三段式结构：

1. 顶部搜索区：Logo、搜索框、搜索按钮、登录入口、购物车入口。
2. 首屏运营区：左侧分类导航、中间活动 Banner 和推荐卡片、右侧用户信息卡片。
3. 商品瀑布区：展示“猜你喜欢”商品列表，点击卡片进入商品详情。

首屏布局如下：

```text
顶部搜索栏
  ↓
频道导航
  ↓
左侧分类导航 + 中间活动区 + 右侧用户卡片
  ↓
猜你喜欢商品列表
```

## 四、路由规划

第一阶段只实现两个页面：

1. `/`：首页，展示分类、活动入口、商品列表。
2. `/product/:id`：商品详情页，展示商品基础信息、SKU、价格和后续加购入口。

后续再扩展：

1. `/cart`：购物车。
2. `/order/confirm`：确认订单。
3. `/orders`：我的订单。
4. `/seckill`：秒杀活动页。

## 五、接口规划

当前前端先预留以下接口：

1. `GET /api/categories`：查询商品分类。
2. `GET /api/products`：查询商品列表，支持 `categoryId`、`keyword`、分页等参数。
3. `GET /api/products/{productId}`：查询商品详情，后端后续接入 Redis 缓存。
4. `GET /api/products/hot`：查询热门商品，后续实现。

第一版首页可以先使用前端静态数据，保证 UI 和交互结构稳定；后端接口增强完成后，再切换为真实接口。

## 六、组件拆分

1. `SiteHeader`：顶部搜索栏和入口区。
2. `CategorySidebar`：左侧分类导航。
3. `UserPanel`：右侧用户信息卡片。
4. `ProductCard`：商品卡片。
5. `HomeView`：首页页面编排。
6. `ProductDetailView`：商品详情页。

组件职责保持清晰，避免首页页面文件越来越大。

## 七、商品详情缓存设计

商品详情接口是高频读接口，适合后端接入 Redis 缓存：

1. 查询商品详情时，先查 Redis。
2. Redis 命中，直接返回商品详情。
3. Redis 未命中，查 MySQL 并写入 Redis。
4. 空结果可以短 TTL 缓存，降低缓存穿透风险。
5. 热门商品可以设置更长 TTL，后续再配合缓存预热。

第一阶段只做商品详情缓存，不急着做复杂缓存一致性。商品数据暂时由 SQL 脚本维护，变更频率低，可以接受较简单的缓存失效策略。

## 八、开发顺序

1. 创建 `mall-frontend` 前端工程。
2. 完成首页静态布局。
3. 完成商品详情页静态结构。
4. 补后端商品列表增强和商品详情缓存。
5. 接入真实接口。
6. 引入 OpenAPI 类型生成。

## 九、当前边界

本阶段不做后台管理接口，不做复杂搜索，不做购物车和秒杀落地。后台商品、分类、库存数据继续通过 SQL 脚本维护，前台优先围绕客户浏览链路推进。
