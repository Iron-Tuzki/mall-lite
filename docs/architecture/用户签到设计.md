# 用户签到设计

## 一、需求范围

用户个人主页展示本月每日签到记录，点击“今日签到”后点亮当天记录。

本阶段只做个人主页轻量能力：

1. 今日是否已签到。
2. 本月累计签到天数。
3. 当前连续签到天数。
4. 本月每日签到热力图。
5. 今日签到幂等处理。

年度热力图、积分奖励、补签卡和签到排行榜后续单独扩展。

## 二、Redis Key 设计

使用 Redis Bitmap 存储用户每月签到记录。

```text
mall:user:sign:{userId}:{yyyyMM}
```

示例：

```text
mall:user:sign:1:202605
```

每日 offset 规则：

```text
offset = dayOfMonth - 1
```

例如：

```text
5月1日  -> offset 0
5月28日 -> offset 27
```

## 三、接口设计

### 1. 今日签到

```text
POST /api/users/sign-in
```

依赖登录态，不需要前端传 userId。

返回当前签到概览：

```json
{
  "todaySigned": true,
  "monthSignedCount": 14,
  "continuousSignedDays": 1,
  "year": 2026,
  "month": 5,
  "daysInMonth": 31,
  "signedDays": [1, 2, 4, 5, 28]
}
```

### 2. 查询个人主页签到概览

```text
GET /api/users/sign-in/profile
```

返回字段同上。

## 四、幂等设计

Redis Bitmap 的 `set(offset, true)` 会返回旧值：

1. 旧值为 `false`：说明今天首次签到。
2. 旧值为 `true`：说明今天已经签到，本次请求直接返回当前概览。

这样重复点击、前端重试、网络抖动都不会重复写入奖励。后续如果接积分，需要只在旧值为 `false` 时发放积分。

## 五、连续签到计算

连续签到从今天往前倒推：

1. 今天已签到：从今天开始向前统计连续的 `1`。
2. 今天未签到：从昨天开始向前统计连续的 `1`。

这样用户还没签到时，也能看到截至昨天的连续签到天数；用户今日签到后，连续天数会立即增加。

## 六、后续扩展

1. 签到积分：首次签到时写积分流水。
2. 连续奖励：连续 7 天、15 天、满签奖励。
3. 年度详情页：按 12 个月 Bitmap 汇总年度热力图。
4. 防刷控制：对签到接口增加 Redis 限流。
5. 运营任务：签到后触发 MQ 消息异步发放优惠券。
