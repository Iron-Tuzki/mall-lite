# 测试种子数据说明

## 一、设计目标

为了减少集成测试中反复创建用户、地址、商品、SKU 和库存的样板代码，项目提供一份固定测试种子数据脚本：

```text
scripts/mysql/005_test_seed_data.sql
```

该脚本可以重复执行，适合本地开发和接口测试。

## 二、固定数据

| 类型 | 固定值 |
| --- | --- |
| 测试用户 ID | `900001` |
| 测试用户名 | `seed_order_user` |
| 测试地址 ID | `900001` |
| 测试分类 ID | `900001` |
| 测试商品 ID | `900001` |
| 测试商品编码 | `SEED-P-001` |
| 测试 SKU ID | `900001` |
| 测试 SKU 编码 | `SEED-SKU-001` |
| 测试 SKU 单价 | `199.00` |
| 测试库存 | `available_stock = 1000, locked_stock = 0` |

测试代码中的 `TestSeedData` 常量类和该脚本保持一致。

## 三、使用方式

本地数据库初始化或重置后，执行：

```bash
mysql -uroot -p09ijklmn mall_lite --execute="source scripts/mysql/005_test_seed_data.sql"
```

执行后，订单和支付相关集成测试可以直接复用这些基础数据，不需要每个测试方法都插入用户、地址、商品和库存。

## 四、注意事项

1. 集成测试使用 `@Transactional`，测试中的库存锁定、订单创建和支付变更会在测试结束后回滚。
2. 脚本会把固定 SKU 的库存重置为 `1000/0/0`，如果你在 Swagger 手动测试后想恢复基础数据，可以重新执行该脚本。
3. 固定 ID 使用 `900001`，避免和普通自增测试数据混在一起。
